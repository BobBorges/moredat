Quick Start Guide

{% translate 'There are two ways to use MOREDAT. If you're just exploring the set of programs for the first time, your best bet is to run MOREDAT on localhost — this means that you run the site on a single machine. Django (the framework MOREDAT is written in) come, by default, with a mock server to test and debug applications on a single computer before they are deployed over the internet. You probably want to use MOREDAT on local host while you make and test customizations.' %}

{% translate 'Once you're satisfied with the customization of MOREDAT, or the addition of MOREDAT apps to another Django project, it's time to put your project "in production", that is to deploy it to the public internet.' %}

Localhost

  • Check / install recent python version
  • Create python virtual environment and install dependencies
  • Run development server

    python manage.py runsslserver

  • Navigage to MOREDAT in a web browser. Either

    https://localhost:8000

    or

    https://127.0.0.1:8000

    Note that it's probably required to explicitly enter https since we run MOREDAT in an ssl development server. Your browser will probably complain about the lack of a valid certificate, but that's ok.

In production

Deploying a website is perhaps a little too vast a topic to describe in detail here, but some points to think about:

  • Decide on and install web server software, e.g. Apache2
  • Decide on and install database software, e.g. MySQL (The sqlite database that MOREDAT comes with is meant for development only --- it's probably too slow for a site in production)
  • Create a python venv (if the hardware will do more than run the MOREDAT instance) and install the required dependencies (or just install the required dependencies)
  • Configure webserver (this will depend on which software you chose)
    • Add project files to a suitable place on the server
    • Update ALLOWED_HOSTS in the settings.py file
    • Edit wsgi.py or asgi.py, depending on your needs (point to your venv if you made one, not system python)
  • Update the settings.py file to point to your new database
  • Makemigrations and migrate the new database (you lose database entries from the development version / sqlite database. If you need them dump them to json first and import them to the new DB after tables are created.)
  • Point your domain name to the server and install an ssl certificate
  • Check if you can get to your instance of MOREDAT at your domain name
  • Turn off DEBUG in settings.py
