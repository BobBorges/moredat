{% translate 'Django' %}

Django ({% translate 'from' %} Wikipedia) {% translate '“is a Python-based free and open-source web framework that follows the model–template–views (MTV) architectural pattern. It is maintained by the Django Software Foundation (DSF), an American independent organization established as a 501(c)(3) non-profit. Django's primary goal is to ease the creation of complex, database-driven websites. The framework emphasizes reusability and 'pluggability' of components, less code, low coupling, rapid development, and the principle of don't repeat yourself. Python is used throughout, even for settings, files, and data models. Django also provides an optional administrative create, read, update and delete interface that is generated dynamically through introspection and configured via admin models.”' %}

{% translate 'How it works (in brief)' %}

{% translate 'When you, as a user, visit a website, you send a request via your browser to the server hosting the website. When you visit a Django site, Django will first try to match the url of your request to a list of pre-defined urls that have been configured to work with the site. Typically urls are defined in a Django project (within the App subdirectories) in files called' %}

  • urls.py
{% translate 'When the request url matches one of the patterns defined in urls.py the specified views function is called.' %}

{% translate 'The views functions that are called after a url pattern match are defined in' %}

  • viwes.py
{% translate 'Views functions are responsible for fetching and processing data from the underlying database, processing that data and passing it to templates, which are rendered as html in the browser.' %}

{% translate 'Template files are a special type of html —Django Template Language— typically stored within the relevant app, e.g.: example_app/templates/example_app/example_template.html.' %}

{% translate 'Database tables are defined as class objects in' %}

  • models.py

{% translate 'Other stuff:' %}

  • Settings: settings.py within the main app directory – various setting definitions are defined here. Perhaps most important is that apps need to be listed under INSTALLED_APPS.
  • Management: manage.py in project root. All sorts of project management can be accomplished with this file.
  • Admin: admin.py in app directories. These files define what data from the App's models are displayed in the Admin pannel.
