Test User Audio

Motivation

If you're going to rely on end users to record themselves, it would be prudent to have some way to at least try and get them to test that their microphone is functional. This app does just that

How it works?

The user has a page with audio controlls on it where they can test their microphone before starting a task that records audio. Tasks that record audio redirect to this app before the "start" button becomes active.

Components

Views and Templates

  • test_audio()

    • This View renders the audiotest.html template which displays a short text describing how to test the microphone, audio controls for actually testing the microphone, and links for suggestions by browser if the audio test isn't working.

  • voice_request()

    • This is a function to save the audio from a user test. Currently it is not implemented, but the researcher could, for instance, add a button to the template to save the audio test, which would send the audio blob to this function.

URLs

Paths and URL names are transparently linked to the view functions by naming conventions.

Static files

There's a functional Javascript file at static/testuseraudio/audiotest.js that handles the recording and playback of audio test. By default these audio blobs aren't saved to the server.
