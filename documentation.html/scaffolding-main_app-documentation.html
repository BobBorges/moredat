Scaffolding: main app

Rationale

This is where all the Django / MOREDAT infrastructure magic happens. The main app defines the root of the project's URLs, the main landing views / pages (those that are nat part of another app), overall settings for the whole program, including which apps are 'active', and which tasks are/can be assigned to users.

Components

Models

  • AcknowledgedContribution

    • This Model has a "person" and a "reason" field, such that contributors to MOREDAT are acknowledged by name in the "person" field and for what contribution in the "reason" filed. Acknowledgments are listed on the "Credits page" (view: credits_view(); template: credits.html

  • CitedReference

    • This Model has fields "item_title", "creators", "production_date", "persistent_id", and "full_reference". It is intended that published works which contributed to the design or motivation of fields will be listed here. CitedReference objects are also listed on the "Credits" page.

  • Tasks

    • The tasks Model is a database table that contains all the tasks designed by the researcher. Unlike the other two models in the main app, the Tasks Model is necessary for the functionality of MOREDAT. In the case when MOREDAT apps are cherry picked to an existing website, the Tasks Model must be replicated somewhere in the project structure of that site.

    Fields:

    • task: name of the task
    • url_name: URL to task
    • app_name: name of app task is built on
    • model_name: name of Model the task is built on
    • url_arg: (optional) variable second component of task URL

Views and Templates

The Views and templates from the main app render landing pages for the scaffolding and elicitation apps' basic descriptions. Views in the main app are most basic, consisting of a function that simply returns a template â€“ with the exception of the credits_view() none of the main apps make any calls to the database. Function names are transparently linked to the main app's templates by their file names.

URLs

In the main app, the urls.py file defines the urls for the scaffolding site, elicitation apps' descriptions, and paths to the actual elicitation apps. Additionally, as the root url specification, this file also defines paths to static and media files, to the admin pages, and for internationalization (translation infrastructure, i18n).

Settings

The settings.py file defines a number of global settings for the MOREDAT project, including:

  • The base directory (folder) for the entire project
  • The hosts allowed to access the site (set to localhost, but for a site on the public internet, the setting needs to be changed)
  • Installed apps: all the apps and installed pip packages used need to be listed here
  • Root URL configuration
  • Database configuration
  • Template configuration
  • Internationalization configuration
  • Media and static file director configuration
