Elicitation: Free Media Upload

Rationale

Sometimes you might like to allow your research participants to upload files that are not part of the apps, whether pictures or audio/video. Perhaps you want them to have the freedom to decide what is important that the researcher sees, or perhaps you as the researcher give little assignments to your participants "Go record your grandma telling you how to cook X in language Y", or "Record you and your friends having a casual conversation in language Y". This app allows you to accept such files.

How it works

The app is a pretty standard file upload page. A user selects a file and provides minimal metadata. Once they click "Submit", the file is transferred to the host server and a metadata entry is created as a database model instance.

Unlike the other elicitation apps, there is no specific functionality to assign uploads as a task. The researcher could communicate what users should upload on their profile page. For instance by adding an "if user.research_group == X" condition followed by a description of the researcher's request for uploads.

Components

Models

There's just one model UploadMetaData() with basic metadata fields for each upload.

Views and Templates

There are two views in the Free Media Upload app:

  • freemediaupload_view()

    • This View displays the upload.html template, which renders a simple upload form (freemediaupload/forms.py UploadForm()).

  • thx_view()

    • This View is a confirmation message after submitting a successful Free Media Upload form. It renders thx.html

Forms

There's just one form freemediaupload/forms.py UploadForm(), which is a model form corresponding to UploadMetaData()

URLs

URL paths and corresponding View functions are transparently linked by the variable names.
